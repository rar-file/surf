# Migration Plan: Add Tavily as configurable search provider alongside DuckDuckGo

**Unit ID:** surf-duckduckgo-add-tavily
**Search Provider:** DuckDuckGo (ddgs / duckduckgo_search)
**Migration Strategy:** additive

## Objective

Introduce Tavily as an opt-in search backend in AISearch. When TAVILY_API_KEY is present (or --search-provider=tavily is passed), use Tavily for web_search and news_search; otherwise fall back to the existing DuckDuckGo path. Keep all existing DuckDuckGo code and env vars untouched.

## Implementation Steps

1. Add `tavily-python>=0.3.0` to requirements.txt
2. In core/ai_search.py: add a TavilyBackend helper that wraps TavilyClient.search(); update AISearch.__init__ to accept provider='auto'|'tavily'|'duckduckgo' and select the backend based on provider arg or TAVILY_API_KEY env var presence
3. Keep existing ddgs/DDGS search() and news_search() code as the DuckDuckGo branch; route AISearch.search() and AISearch.search_async() through the selected backend
4. In core/surf.py: add --search-provider CLI argument; read TAVILY_API_KEY from env; instantiate AISearch with the resolved provider; no changes to do_search/do_research call signatures needed
5. In core/mcp_server.py: update get_skill() to pass provider from env/config; update web_search Tool description to note Tavily is used when key is present

## Files to Modify

### `requirements.txt`
- Add tavily-python dependency alongside existing ddgs/playwright deps

### `core/ai_search.py`
- Central search abstraction — add a TavilyBackend class and a provider-selection factory inside AISearch.__init__ that checks TAVILY_API_KEY; existing DuckDuckGo code stays as the default branch

### `core/surf.py`
- CLI entry point — expose --search-provider flag and/or honour TAVILY_API_KEY env var; pass provider choice to AISearch; update do_search() and do_research() to use the configured backend

### `core/mcp_server.py`
- MCP server — passes through to AISearch; update get_skill() to forward the provider setting so MCP clients also benefit from Tavily when the key is configured

## Dependency Changes

- Add tavily-python>=0.3.0 to requirements.txt

## Environment Variable Changes

- Add TAVILY_API_KEY (new, optional — enables Tavily backend when set)
- Keep existing DDGS-related setup as-is (no removal)

## Notes

DuckDuckGo requires no API key and remains the zero-config default. Tavily activates automatically when TAVILY_API_KEY is set, or explicitly via --search-provider=tavily. The page-fetch path (Playwright) is unaffected by this migration — Tavily only replaces the DDGS query step, not the HTML fetching.
